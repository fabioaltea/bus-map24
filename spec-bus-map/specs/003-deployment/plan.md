# Implementation Plan: Production Deployment

**Branch**: `003-deployment` | **Date**: 2026-04-24 | **Spec**: [spec.md](./spec.md)

---

## Summary

Deploy BusMap24 frontend (Vercel) and backend (Railway). Backend requires a worker entry point, a programmatic migration runner, CORS env-var config, and Railway/Vercel config files. PostgreSQL runs as a custom Docker service (`postgis/postgis:17-3.4`) on Railway to preserve PostGIS support.

---

## Technical Context

**Language/Version**: TypeScript 5 / Node.js 22 LTS (API + Worker), React 18 / Vite 5 (FE)  
**Primary Dependencies**: Fastify 4, Drizzle ORM 0.31, BullMQ, @fastify/cors  
**Storage**: PostgreSQL 17 + PostGIS 3 (Railway Docker), Redis (Railway addon), Railway Volume (PMTiles)  
**Target Platform**: Vercel (FE), Railway (API, worker, DB, Redis)  
**Performance Goals**: API p95 < 500 ms; FE Lighthouse ≥ 90  
**Constraints**: PostGIS required; workers must be long-lived (no serverless); migrations must run before traffic

## Phase 1: Backend changes

### 1.1 `src/worker.ts`

Standalone process that imports and starts `catalogSyncWorker` and `feedDownloadWorker`. Handles graceful shutdown on `SIGTERM`.

### 1.2 `src/db/migrate.ts`

Calls Drizzle's `migrate()` programmatically with the compiled migration files. Used as `node dist/db/migrate.js` in Railway start command.

### 1.3 CORS (`src/app.ts`)

```ts
await app.register(cors, {
  origin: process.env.CORS_ORIGIN === '*' || !process.env.CORS_ORIGIN
    ? true
    : process.env.CORS_ORIGIN.split(',').map(s => s.trim()),
})
```

### 1.4 Health check

Add `GET /healthz` route: queries `SELECT 1` on DB and pings Redis; returns `{ status, db, redis }`.

### 1.5 `railway.json`

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": { "builder": "NIXPACKS" },
  "deploy": {
    "startCommand": "node dist/db/migrate.js && node dist/server.js",
    "healthcheckPath": "/healthz",
    "healthcheckTimeout": 30,
    "restartPolicyType": "ON_FAILURE"
  }
}
```

### 1.6 `package.json` additions

```json
"worker": "node dist/worker.js",
"db:migrate:prod": "node dist/db/migrate.js"
```
